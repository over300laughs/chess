Documentation

  • Game grid that uses CSS and is generated by JavaScript

    A style sheet is included in this project and sets positioning for piece images based on classes named pawn, queen, etc. colors of pieces are also distinguishable using a CSS class named w or b.

  • User input in the form of text, select options, and/or buttons

    Text boxes, a form, and buttons are used to enter the names of the players. A button is also used as well to load a saved game. (See right side of the board.)

  • User input in the form of mouse clicks

    Clicks are employed to allow players to select and deselect pieces. Essentially all player input is done using the click.

  • Dynamic modification of HTML using innerHTML or DOM functions

    The code innerHTML is used any time a piece is moved in the move function. Its also used in the function loadGame(). (Last function in model.js)

  • Use of XMLHttpRequest to load JSON or XML game data that is displayed on the grid page

    This is used in the function loadGame() (Last function in model.js). An XmlHttpRequest is made and the XML (savedGame.xml) is parsed and loaded to a saved game. The color of the player is given for whp's turn it is and the users can pick up where they left off.

  • Incorporates HTML 5 audio tags, video tags, canvas, or local storage

    Some music was added using the html audio tags. The tags are located in js-board.html on line 11. The javascript is handled in the function playMusic() located on line 15 of chess.js

  • Game logic code

    Game logic is mostly found in model.js. Functions for each of the possible moves are seperated into functions that are named showQueenMoves() or showBishopMoves() and so on. These functions find all possible moves for the selected piece and then pass array with the possible moves.
